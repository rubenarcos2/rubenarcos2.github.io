    Integrated project at the end of the Higher Technician in Multi-platform Applications Development

    The main purpose of the application has been to demonstrate the implementation of the knowledge acquired throughout the training cycle of multiplatform application development. I have focused the application on the knowledge acquired on the management of user access privileges to different elements (or modules as I will refer to throughout the document) of the application, the management of hierarchical users in different groups (called roles) and the administration by an authorised user. No less relevant has been the application of knowledge in the design and management of relational databases and their implementation platforms. Another factor taken into account has been the security of the environment of use and of the application, implementing functionalities such as the physical blocking of the terminal, the encryption of sensitive data, or the prevention of the use of elements sensitive to the injection of automated information (cracking or brute force attacks).

    Executable and source code

    1. Carry out the installation.
    2. Execution of GesMerca
    3. The default credentials are:
      • Administrator: admin/admin
      • User: ruben/123456
      • Terminal lock code: 5555
    4. In the configuration window, once logged in as administrator, you can change the way to perform the initial login, also the personalised configuration for a specific user, in the same window. The privileges will be detected automatically.
    5. The connection to the database is configured to a test copy in the cloud by default. However, it is possible to change the configuration in a local file (GesMerCa.exe.config).
    6. Once the installation is done, if you find that you are having database connection problems, make the following change:
      • In the installation path of the application, modify the GesMerCa.exe.config file, in line 13 and add SSLMode=0 at the end;
